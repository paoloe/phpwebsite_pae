What is a Webmaster?

In this web-based tutorial for PHP, there will be a lot of reference to a 'Webmaster'. It is important that it is defined as it plays an important part in the explanation aspect of this tutorial.

Webmaster, in this context are the people and or person that manages the webserver. They will often be referred to in this tutorial. Their varying skill level, experience, company and company size affect what features are suitable for them or their situation, it is important that this is considered as not all the features might not be suitable for all of them.

Webmasters manage web pages, sites and applications. They coordinate the design, development, deployment and maintenance of a company's online presence. (jobisjob, 2020).

PHP (Hypertext Preprocessor)

What is PHP?

Sklar (2016) describes PHP to be a programming language which is run on a webserver that is typically accessed by many users at one time. It is a server-side language as all the processes it carries out are within in the server, this is best illustrated in Figure 1. (Sklar, 2016).

What are the common uses of PHP?

PHP is commonly used for creating a dynamic website. Companies such as Facebook, Wikipedia and Wordpress makes use of PHP (CyberCraft, 2020), one of features of PHP and one of the reasons why such large companies make use of it is, its ability to interact with the underlying data that a server has access to, be it that data is on a disk drive or a database.

Facebook for example has billions of users, as a result of this, records of the users are kept in a database. PHP is great for interaction with databases and webservers it can retrieve data from the database and apply it to a template, this can be seen in Figure 2, highlighted in red are the different bits of information retrieved from their database to then be displayed on to a template (the template being the page).

Within the example of Figure 2 there are multiple processes that PHP carryout, it includes the reading the request, processing the request, contacting the relevant data layer and finally sending the response for the server to respond to the client.

How is PHP Useful to a Webmaster?

Mentioned briefly earlier in this report, PHP is a server-side language, what is a server side language? Sam OverMars' (2020) answer to this was It is a programming language that is designed to run on a server. Because it is tailored for this reason it has benefits such as, it allows us to efficiently deliver information tailored for individual users and thereby create a much better user experience.

One of the main reasons as to why PHP is useful to a Webmaster is the ability in which they are granted to create a dynamic website. It allows for a fully customised experience for each of the users of the website, this can be seen demonstrated in websites such as Facebook, YouTube and Amazon where the more the user utilizes these websites, the more that the site becomes personalised to their specific use.

Part of how these websites are able to deliver a unique experience for each user, is the capability that PHP allows for data analysis, PHP can be used to refine responses based on analysis of this data. (Mozilla, 2020) Given the example of Facebook marketplace, the items for sale that a user will see more of will be based on their previous searches and items that they've looked at.

Possible Issues of Using PHP to Collect Form Data

Note, the code in the previous tutorial does not consider security features. It is not safe from issues such as SQL injection (SQL injection, is malicious code that is 'injected' into your server to either cause damage or get hold of confidential data information).

A common way of people being able to do this is through forms on websites. There have been many famous and severe cases that have occurred to large public and private companies whereby people have defaced websites, exposed confidential information and even a company like Starbucks has been a victim of this issue, whereby a SQL injection allowed access to the Starbucks accounting database (codecurmudgeon, 2020).

There are measures that can be taken to improve the security of a PHP page to prevent SQL injection. Use prepared statements and parameterized queries. (PeeHaa, 2019) This answer is one of the most popular answers on Stack Overflow. It simply means, any code that is sent to the server database uses a format that has been prepared by the webmaster and the input of the users are parameters. This means that the user input is only what is required from them for example, if the input was a first name, this would be a parameter slotted into a pre-prepared statement.

As PHP is a server-side language, it is important that security is always considered as it is a broker between the client and the server in a request and send process. Any webmaster who chooses to use PHP (or any server languages) needs to prioritise what security measures to incorporate to keep their server safe.

Tutorial 3: PHP File Handling / Management

It is important that if a site were to give the ability to the user of being able to access, read and or edit files on the server, that it is done with extra care.

This feature could be useful to webmasters that are part of a company that has services that upload paid publications to their website. PHP allows the permitted users access to the publications that they are subscribed to.
